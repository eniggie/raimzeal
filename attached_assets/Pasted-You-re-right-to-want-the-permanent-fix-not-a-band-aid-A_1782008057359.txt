## The decision: remote version source + auto-increment

Your instinct ("submit immediately and update itself") = exactly what `appVersionSource: "remote"` + `autoIncrement: true` does. EAS keeps the build number on **its own servers** and bumps it automatically every build — so a collision like today's becomes impossible.

**Why not the lazy version** (just flip `autoIncrement: true` and leave it "local"): on EAS *cloud* builds, "local" auto-increment can't write the new number back to your repo, so it re-reads the same base and produces duplicates again. **Remote is the only setup that truly self-updates.** This is also Expo's own recommended config.
